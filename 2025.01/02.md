# 낙관적 업데이트

대표적인 예시는 "좋아요" 기능

서버 응답이 성공일 것이라고 가정하고 UI를 먼저 업데이트하는 것

사용자 입장에서 좋아요를 클릭하면 곧바로 UI 인터랙션이 발생하기 때문에 더 효과적인 UX 달성 

만약에 실패할 경우를 대비해서 롤백 로직을 필요하기도 함 (서버 응답이 실패 => 원래 UI로 되돌림)

결제, 회원가입 등 중요한 기능에는 사용을 지양하는게 좋음
